KB Prep Tool

把本地拿到的 PDF、Office、网页、字幕、笔记、电子书、代码资料，转换成结构清晰、正文细节完整、污染内容可追溯的 Markdown。

查看 GitHub 安装命令
detectPDF / Word / PPT / 字幕 / 网页 / 电子书
convert尽量无损转 Markdown，保留图片和表格
blockify按章节、段落、代码、字幕块切分
clean去广告、二维码、口播废话，不删教程细节
reportcleaned.md + discarded.md + quality_report.json

一件事做深

它不是 RAG，不抓平台内容，不生成最终 wiki。它只负责把已经在本地的原始资料变成干净 Markdown。

PDF 智能分流

可信文字层走 PyMuPDF；扫描件、图片型、乱码文字层、MOBI 和旧 Office 走 MinerU/OCR。

Office / EPUB 轻量转换

DOCX、PPTX、XLSX 和 EPUB 优先本地解析，保留文本、表格、章节顺序和嵌入图片。

字幕、网页、代码、笔记

字幕去时间戳，HTML 保留链接和图片，代码文件包进 fenced code block，Markdown 保留 Obsidian 结构。

处理流程

长文档不会按页粗暴切开，而是先识别结构，再按块清洗，最后按原顺序拼回。

检测

判断文件格式、PDF 子类型、文字层质量、是否需要 OCR。

转换

输出 `converted.md`，同时复制本地或嵌入图片资源。

切块

按标题、自然段、字幕段、代码块、表格建立 `blocks.jsonl`。

清洗

删除污染内容，保护步骤、参数、Prompt、代码、链接、案例。

验收

生成质量报告；正文细节丢失、CTA 残留、图片缺失都会被拦住。

清洗不等于总结

  • 必须保留：步骤、工具平台、账号设置、参数、链接、Prompt、代码、表格、案例过程、失败经验。
  • 必须清理：二维码、扫码入群、体验卡、购买引导、无关页眉页脚、重复口播、纯广告图片。
  • 不能按关键词误删：如果“平台、账号、社群、扫码”出现在教程案例或规则说明里，会保留上下文。
  • 可追溯：`discarded.md`、`review_needed.md`、`evidence/marketing_pages.md` 带 block id、页码、章节、原因和置信度。

安装到 OpenClaw

插件首次运行会在自己的目录创建 `.kbprep/venv`，依赖只安装在插件环境里，不污染系统 Python。

openclaw plugins install git:github.com/maple192600-LI/openclaw-kbprep
openclaw gateway restart
openclaw plugins inspect openclaw-kbprep --runtime --json
