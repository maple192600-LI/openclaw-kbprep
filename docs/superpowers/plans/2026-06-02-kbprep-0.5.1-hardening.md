> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the verified KBPrep v0.5.0 hardening issues without doing a broad repository rename or large architectural rewrite.

**Architecture:** Keep changes inside the existing Python worker, TypeScript worker bridge, and tests. ### Task 2: Worker Stdin Backpressure

**Files:**
- Modify: `src/worker.ts`
- Modify: `src/worker.test.ts` if a small unit-style regression is feasible without slow process mocking

- [ ] **Step 1: Add helper in `src/worker.ts`**

Add a focused helper near `callWorker`:

```ts
async function writeStdin(stream: NodeJS.WritableStream, input: string): Promise<void> {
  await new Promise<void>((resolve, reject) => {
    const onError = (err: Error) => {
      cleanup();
      reject(err);
    };
    const onDrain = () => {
      cleanup();
      stream.end();
      resolve();
    };
    const cleanup = () => {
      stream.off("error", onError);
      stream.off("drain", onDrain);
    };

    stream.once("error", onError);
    if (stream.write(input)) {
      cleanup();
      stream.end();
      resolve();
      return;
    }
    stream.once("drain", onDrain);
  });
}
```

Replace:

function parseFirstJsonArray(text: string): unknown[] | null {
  const fencedMatches = [...text.matchAll(/```(?:json)?\s*([\s\S]*?)```/gi)].map((match) => match[1]);

## Acceptance Criteria

- CTA keyword constants are deduplicated.
